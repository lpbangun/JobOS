## Variant: Hybrid Cockpit (practical surface)

### Design stance
A + B + C: pipeline/today structure, optional agent pair pane, and `:` command escape hatch — without painting every JobOS feature as permanent chrome.

### Primary (always)
- Today / priority brief
- Pipeline board + Jobs table (two views of same core)
- Selected job context + next actions
- Review queue (human gate)
- Agent/run status (rail + system)

### Secondary (drawer on demand)
- Network, Outreach, Answers, Discovery sources, Funnel/weekly

### Agent / command (long-tail)
- Multi-step pursue with steering
- Research synthesis, bulk triage, weekly narrative
- Full CLI parity via `:` bar
- Browser/rare config not given permanent panels

### Interaction
- `1-5` primary views
- `a` toggle agent pane
- `p` pursue, `d` daily
- `n/o/q/s/f` secondary drawers
- `:` focus command bar
- `j/k` move job selection

### Trade-offs
- Strong at: daily job-search cockpit without losing agent power
- Weak at: maximum density of every automation/browser detail (intentionally deferred)
