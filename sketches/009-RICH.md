# Rich shells (009–010)

Response to “picked shells feel empty”: keep lazygit-style stable list|detail, but put **real JobOS signal** in always-on surfaces.

## What’s always visible now

| Region | Content |
|--------|---------|
| Header counts | open / high-fit / due / drafts / interviews |
| Priority strip | 4 today cards (due, interview, new high-fit, source failure) |
| Jobs list | **A:** cards with fit, stage, next, proofs/drafts/path signals · **C:** dense table (id, role, co, fit, stage, next, src) |
| Detail | summary · fit/mode · narrative · **next** · proof chips · warm path · artifacts · pursue stage strip · actions |

## Toggle

- **Agent** (`a`) — side stream
- **Review** (`r`) — draft queue under list
- **Log** (`l`) — event tail (C energy, off by default)
- **Cmd** (`:`)
- **Profile**

## Overlay

- Network, Docs reader, Answers match, Discovery health, System

## Why earlier felt empty

006–008 only showed title + next + buttons. Cut C2 fit, proofs, path, artifacts, stages, today strip, and counts — so the shell had layout but no *job-search substance*.
